# 内容监测分析仪表盘

一个基于飞书数据源的内容分析小工具。架构核心是 **「我（WorkBuddy 大模型）作为数据管线」** + **「静态网页作为工具」** 两层：

- **第 1 层 · 自动数据管线（定时任务跑我）**：定时拉取飞书多维表格 → 我做大模型分析（爆款特征、选题方向、创意洞察）→ 生成 `data/content_data.json` + 一份 `data/insights.json` 洞察报告 → 提交回仓库。
- **第 2 层 · 网页工具（静态前端）**：数据浏览 + 四大板块可视化；**即时爆款率评分**由前端脚本匹配（输入即出分）；**深度分析**在对话里由我（大模型）完成，网页只负责把上下文一键复制给你。

> 为什么深度分析不走"网页→写回 git→我再拉→回写"的异步环？因为纯静态网页无法实时调用大模型。深度分析本就该在我们的**对话里**发生：我随时带着飞书全量数据上下文，出的报告最丰富，还能直接存回仓库变成站点上的"洞察报告"。

## 核心功能

### 数据浏览（四大板块 + 总览）
- **总览**：核心指标、发布频率与表现趋势、多账号横向对比
- **内容策略**：最佳发布时段、内容形式 / 主题 / 情绪分布
- **爆款规律**：爆款 vs 普通雷达图、内容类型 ROI Ranking、Top 10% 自动标记、爆款共性
- **活动监测**：Campaign 集中爆发识别（频率提升倍数）
- **用户分析**：评论质量分布、品牌回复率 / 回复时长

### 内容库 / 灵感库
- 按账号、形式、主题、情绪、活动标签、关键词筛选
- 快速定位 Top 10% 高表现内容

### AI 洞察报告（大模型产出）
- 由定时任务中的我自动生成：`data/insights.json`
- 包含：爆款共性特征、选题方向建议（带 why + example）、关键洞察、风险提示、下一步行动
- 打开网页「AI 洞察」tab 即可查看

### AI 选题预测（两层）
- **第 1 层（即时）**：前端脚本基于历史爆款特征给「爆款概率」评分，输入即出，无需等待
- **第 2 层（深度）**：点「复制给 AI 做深度分析」→ 把上下文复制到剪贴板 → 发给 WorkBuddy（我）→ 我基于飞书全量数据出详细报告（概率评估 + 形式/时段/情绪建议 + 修改方向 + 落地标题）

## 快速开始

### 本地预览

```bash
cd content-analytics-dashboard
python3 -m http.server 8080
# 打开 http://localhost:8080
```

未配置飞书密钥时，会自动使用 `data/sample_data.json` 作为演示数据，`data/insights.json` 为示例洞察报告。

## 数据源：飞书多维表格

### 1. 创建飞书自建应用
1. 进入 [飞书开放平台](https://open.feishu.cn/) → 创建企业自建应用
2. 记录 **App ID** 和 **App Secret**
3. 开启权限：`bitable:record:read`

### 2. 多维表格字段（建议）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 账号 | 文本 | 品牌主号 / 副线账号A / 副线账号B |
| 平台 | 文本 | 小红书 / 抖音 / 微博 |
| 发布时间 | 日期时间 | ISO 8601，如 2026-07-10T18:00:00+08:00 |
| 内容形式 | 单选 | 视频 / 图片 / GIF / 投票 |
| 主题标签 | 多选/文本 | 产品介绍 / 活动 / 红人 |
| 内容情绪 | 单选 | 高级 / 搞笑 / 性感 / 争议 / 普通信息 |
| 活动标签 | 单选 | 无 / 赠礼活动 / 挑战 / 节日 / 联名合作 / 直播 |
| 是否活动 | 单选 | 是 / 否 |
| 活动名称 | 文本 | Campaign 名称，非活动留空 |
| 内容文案 | 文本 | 标题/正文 |
| 曝光量 | 数字 | |
| 点赞数 | 数字 | |
| 转发数 | 数字 | |
| 评论数 | 数字 | |
| 收藏数 | 数字 | |
| 品牌回复数 | 数字 | |
| 平均回复时长_分钟 | 数字 | |
| 赞美 / 提问 / 分享体验 / 开玩笑 / 抱怨 | 数字 | 评论质量维度 |

> 拉取脚本见 `scripts/sync_feishu.py`（在定时任务中被调用，或本地手动跑）。

## 数据管线：定时任务（我作为引擎）

本项目不依赖 GitHub Actions。数据更新由 **WorkBuddy 定时任务** 完成：

- 任务名：`内容监测仪表盘 · 数据管线（拉飞书→大模型分析→产出洞察）`
- 触发：每天一次（可在 WorkBuddy 自动化里调整频率，或手动 Run）
- 动作：
  1. 拉取飞书多维表格（未配置则回退演示数据）
  2. 运行 `js/analysis.js` 计算真实指标
  3. **由我（大模型）生成 `data/insights.json` 洞察报告**
  4. `git commit`（若配置了远程且凭据可用则 `git push`）

启用前需确认：
- 已连接飞书连接器（或配置 `FEISHU_*` 环境变量），否则用演示数据；
- 已连接 GitHub 连接器（或本地有 remote + 凭据），否则只提交到本地仓库，需手动 push 才能更新线上站点。

## 部署

### 方式 A：GitHub Pages
```bash
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```
仓库 Settings → Pages → Source 选 `main` / root。定时任务 push 后站点自动更新。

### 方式 B：CloudStudio / 任意静态托管
把整个目录作为静态站点部署即可（无需后端）。

## 文件结构

```
content-analytics-dashboard/
├── index.html              # 主页面（两层结构）
├── css/styles.css          # 样式
├── js/
│   ├── data.js             # 数据加载器（内容 + 洞察）
│   ├── analysis.js         # 分析引擎（指标计算，被定时任务复用）
│   └── app.js              # 主应用逻辑（含 AI 洞察渲染、选题预测两路）
├── data/
│   ├── sample_data.json    # 示例数据
│   ├── content_data.json   # 飞书同步后的真实数据（定时任务生成）
│   └── insights.json       # 大模型生成的洞察报告（定时任务生成）
└── scripts/
    └── sync_feishu.py      # 飞书同步脚本
```

## 说明

- 即时爆款率评分在浏览器端由脚本完成（快、无需等待）；深度分析必须经过大模型，在对话里完成。
- 所有可视化均为静态前端，无后端依赖。
- 想让我现在跑一次完整管线？直接在对话里说"跑一次数据管线"即可。
